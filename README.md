# playlist-maker-android-TiniakovAndrei
Учебный проект по созданию мобильного приложения


## Задача 1. Вёрстка экрана поиска

- Реализована вёрстка SearchScreen, используя Jetpack Compose и опираясь на дизайн-макет экрана в Figma.

- Реализована логика сброса поискового запроса.

- Для поисковой строки использована compose-функция TextField или OutlinedTextField.

- Логика поиска оставлена пустой. Она будет реализована позже.


## Задача 2. Вёрстка экрана настроек

- Сверстан экран настроек по дизайн-макету из Figma.

- Реализованы кнопки «Поделиться приложением», «Написать разработчикам», «Пользовательское соглашение».


## Задача 3. Переходы между экранами

- Создан enum-класс с перечислением имеющихся экранов.

- Создан класс PlaylistHost — он будет основной сущностью, отвечающей за навигацию (это должен быть @Composable, принимающий на вход NavController).

- Добавлен в PlaylistHost класс NavHost, в котором будут задаваться все экраны (MainScreen, SearchScreen, SettingsScreen).

- Созданы в PlaylistHost методы для перехода на экраны, которые бы использовали NavController и enum-класс из первого пункта.


## Чек-лист самопроверки
✅Навигация в приложении между экранами работает без падений.

✅Вся логика навигации реализована в классе PlaylistHost.

✅По нажатию на кнопку «Search» выполняется переход на экран поиска.

✅По нажатию на кнопку «Settings» выполняется переход на экран настроек.

✅При добавлении новых экранов не забывать обновлять код навигации.
